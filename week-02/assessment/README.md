# Week 2 Assessment — Contact Book Manager

**Due:** Wednesday night (before Week 3, Session 1)  
**Format:** Individual work, submitted via GitHub (same process as Week 1)  
**Not graded** — but Thursday's 30-minute review is built entirely from this, so treat it as required.

---

## What you're building

A **Contact Book Manager** — a console application that stores, searches, updates, and deletes contacts using dictionaries and lists. Each contact is a nested dictionary with name, phone, email, and optional tags.

---

## Structure

Work through these in order:

1. **[Exercises](./exercises/)** — Isolated drills for lists, tuples, dicts, and sets. Do these *before* starting the project.
2. **[Project Spec](./project/PROJECT_SPEC.md)** — The Contact Book Manager requirements. This is the main deliverable.
3. **[Optional Challenges](./challenges/)** — Extra features if you finish early and want more practice.
4. **[Reflection Log](./reflection_log.md)** — Answer these after you're done (helps solidify what you learned).
5. **[Common Mistakes](./common_mistakes.md)** — Read this if you get stuck, or after you submit to see what others trip on.

## Submission checklist

Before you push to GitHub:

- [ ] All exercises in `exercises/` are complete

- [ ] Your code has no syntax errors (run it end-to-end at least once)
- [ ] You've filled out the reflection log
