# Optional Challenges

These are optional extensions for students who finish the core exercises and project early.
They are meant to push your understanding beyond the minimum requirement and make your contact book feel more polished.

## Challenge 1 — Contact Tag Filter
Create a feature that lets the user filter contacts by a tag.

### What to implement
- Ask the user for a tag to search for.
- Print every contact that contains that tag.
- If no contacts match, print: "No contacts found for that tag."

### Why this helps
This makes you practice nested data structures and searching through lists inside each contact record.

## Challenge 2 — Alphabetical Contact Book
Modify your contact book so that it prints contacts sorted by name.

### What to implement
- Use `sorted()` on the contact names.
- Display contacts in alphabetical order.

### Why this helps
This reinforces the idea that lists can be sorted and displayed in a more user-friendly way.

## Suggested approach
1. Finish the required project first.
2. Add one challenge at a time.
3. Test each feature after you add it.
4. Keep your code readable and avoid copying large blocks of logic.
