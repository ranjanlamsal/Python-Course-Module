# Week 3 Assessment

## Overview

This assessment has **5 warmup exercises**, **2 mandatory projects**, **3 optional challenges**, and a **reflection log**.

It is not graded. It is the *only* material next Thursday's 30-minute review will be built from. Treat exercises and projects as required; challenges as optional.

⚠️ **Compressed timeline:** Both projects (Cart Engine + Log Analyzer) are due **Wednesday night**. This is intentional but tight. Start Saturday morning, not Sunday night.

## Timeline

- **Saturday:** Exercises (2-3 hours)
- **Sunday-Tuesday:** Projects 3 & 4 (split your time, don't save one for last)
- **Wednesday:** Final touches, test edge cases
- **Thursday:** 30-min review in class

5. **[`exercise_5_refactor_with_functions.py`](./exercises/exercise_5_refactor_with_functions.py)** — Refactor repetitive code into reusable functions

### Projects (Required)

Two substantial projects that demonstrate Week 3 concepts:

### Reflection Log

## How to Use This Assessment

1. **Start with exercises** (Saturday): They're short and targeted. Do all 5 before moving to projects.
2. **Read both project specs** (Saturday): Understand requirements, sketch data structures.
3. **Work on projects in parallel** (Sunday-Tuesday): Alternate days, or split each day. Don't save one for last.
4. **Test edge cases** (Wednesday): Empty inputs, missing files, malformed data.
5. **Fill out reflection log** (Wednesday): Short answers, honest reflection.

## Common Mistakes

See **[`common_mistakes.md`](./common_mistakes.md)** for anonymized bugs from previous cohorts. Read this *before* starting projects—it will save you hours of debugging.

## Getting Unstuck

If you hit a wall:
1. Re-read the relevant handbook section
2. Run the demo code, modify it, experiment
3. Attend office hours (schedule in syllabus)
4. Ask classmates (explain your approach, not your code)

**Don't stay stuck for more than 30 minutes.** If you're spinning, get help.
